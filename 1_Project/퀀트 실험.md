## 2026-02-21 업데이트 (OpenClaw)

### 1) 현재 상태 요약

- **Phase 0~2 완료**
- **Phase 3 진행 중** (장중 리허설/운영정책 고도화)
- 엔진은 50종목, 15분봉/15분 주기로 운용

### 2) 완료된 항목

#### Phase 0
- 주문 스키마 정합성 (`order_type`/`type`)
- WebSocket 라우트-스트리밍 정합성
- 엔진 `_evaluate_signals()` 실제 연결
- 자동매매 제어 API (`start/stop/status/health/run-once`)
- paper 모드 강제 가드 (`ALPACA_PAPER=true` 필수)

#### Phase 1
- 알림 하드닝
  - risk block, daily-loss block, order failure
- 통합 테스트 보강
- runbook/체크리스트 문서화

#### Phase 2
- kill-switch engage(close-all) 경로 구축
- reconcile endpoint + status 반영
- 실패 시 alert/event 기록 강화
- `/automation` 운영 UI 반영

### 3) 2026-02-21 전략 튜닝 반영

트리거 부족 문제 대응으로 기본 전략 프로필을 완화/가속.

- 타임프레임: **15Min**
- 평가 주기: **15분**
- 최소 신뢰도: **0.45**
- 전략 조합:
  - `MACrossoverFast` = EMA(5/13)
  - `RSIFast` = RSI(7), 35/65
  - `MACDFast` = 8/21/5
- 합의 방식: 전략별 신호를 confidence 가중 합으로 BUY/SELL 결정

관련 커밋:
- `e51cce6` 15분봉 자동평가 + 50종목 유니버스
- `2552ef5` 빠른 멀티전략 + min_confidence 완화

### 4) 최근 리허설 체크 결과

- 엔진 상태: `running`
- 대상 종목: `50`
- run-once 결과: `signals_evaluated=18`, `executions=2`
- 생성 주문 예시: `NVDA`, `V` (buy, accepted)

> 참고: 장외 시간에는 체결이 바로 안 보일 수 있음.

### 5) 남은 일 (Phase 3)

1. 장중 실운영 리허설 1회 이상 + 캡처/로그 증적 축적
2. reconcile 운영 정책 확정 (빈도/재시도/승인 규칙)
3. kill-switch close-all 실패 시 승인/롤백 플로우 문서화
4. 운영 KPI 대시보드(fill/reject/latency/PnL) 정리
